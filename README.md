# Сайт-визитка

Набор файлов для простой статической страницы-визитки, готовой к размещению на GitHub Pages.

Как опубликовать:

1. Инициализируйте репозиторий и закоммитьте файлы:

```bash
git init
git add .
git commit -m "Initial: site-vizitka"
```

2. Создайте репозиторий на GitHub и подключите его как `origin`:

```bash
git remote add origin https://github.com/<your-username>/<repo>.git
git branch -M main
git push -u origin main
```

3. В настройках GitHub (Settings → Pages) выберите ветку `main` и папку `/ (root)`, затем сохраните. Через минуту сайт будет доступен по адресу `https://<your-username>.github.io/<repo>/`.

Примечания:
- Для кастомного домена добавьте файл `CNAME` с доменом и настройте DNS.
- Чтобы скрыть генератор Jekyll (если у вас файлы начинаются с `_`), добавьте пустой файл `.nojekyll`.

Менять содержимое страницы можно прямо в `index.html` и `styles.css`.

Логотипы заказчиков:

- Временно добавлены заглушки в `assets/logos/` (файлы *.svg). Замените их на официальные логотипы компаний.
- Где взять официальные логотипы: сайты самих компаний, разделы «Пресс‑центр/Медиа», или Wikimedia Commons (если доступны с лицензией). При необходимости я могу попытаться найти логотипы и добавить их в репозиторий — подтвердите, нужно ли мне это сделать.
